Operating Systems

An operating system (OS) is the software that manages all hardware and software on a computer. Understanding how operating systems work is crucial for cybersecurity.

What is an Operating System?

The OS is the bridge between you and the computer hardware. It manages:

  • Memory: Allocating RAM to programs
  • CPU: Scheduling which processes run
  • Storage: Managing files and directories
  • Input/Output: Handling keyboard, mouse, display
  • Security: User permissions and access control

Key OS Concepts

1. Processes and Threads

A process is a running instance of a program. Each process has:

  • Its own memory space
  • Process ID (PID)
  • State (running, sleeping, stopped)
  • Priority level
# View running processes
ps aux

# View processes with hierarchy
ps auxf

# Real-time process monitoring
top

2. File Systems

File systems organize how data is stored on disk:

  • ext4 - Default Linux file system
  • NTFS - Windows file system
  • APFS - macOS file system
  • FAT32 - Universal but limited
# Check disk space
df -h

# View file system type
mount | grep "^/dev"

# Check inode usage
df -i

3. Permissions and Access Control

Linux uses a permission system with three types of users:

  • Owner (u) - The file creator
  • Group (g) - Users in the file's group
  • Others (o) - Everyone else

Three permission types:

  • Read (r) - View file contents (4)
  • Write (w) - Modify file (2)
  • Execute (x) - Run file as program (1)
# View permissions
ls -la

# Change permissions (symbolic)
chmod u+x script.sh      # Add execute for owner
chmod go-w file.txt      # Remove write for group and others

# Change permissions (numeric)
chmod 755 script.sh      # rwxr-xr-x
chmod 644 file.txt       # rw-r--r--

# Change ownership
chown user:group file.txt

Security Tip: Never run scripts with 777 permissions in production. This allows anyone to read, write, and execute your files!

Networking Basics

Networking is how computers communicate. Most cyber attacks happen over networks, so understanding networking is essential.

The OSI Model

The OSI (Open Systems Interconnection) model has 7 layers:

  1. Physical - Cables, signals, hardware
  2. Data Link - MAC addresses, switches
  3. Network - IP addresses, routers
  4. Transport - TCP/UDP, ports
  5. Session - Connection management
  6. Presentation - Data formatting, encryption
  7. Application - HTTP, FTP, SSH, DNS

IP Addresses

IP addresses identify devices on a network:

IPv4

  • Format: 192.168.1.1 (four octets, 0-255 each)
  • 32 bits total (~4.3 billion addresses)
  • Private ranges:
    • 10.0.0.0/8 (10.0.0.0 - 10.255.255.255)
    • 172.16.0.0/12 (172.16.0.0 - 172.31.255.255)
    • 192.168.0.0/16 (192.168.0.0 - 192.168.255.255)

Subnetting

Subnetting divides networks into smaller networks:

192.168.1.0/24 means:
- Network: 192.168.1.0
- Subnet mask: 255.255.255.0
- Usable IPs: 192.168.1.1 - 192.168.1.254
- Broadcast: 192.168.1.255
- Total hosts: 254

TCP vs UDP

TCP (Transmission Control Protocol)

  • Connection-oriented (3-way handshake)
  • Reliable delivery with error checking
  • Used by: HTTP, HTTPS, SSH, FTP

UDP (User Datagram Protocol)

  • Connectionless (fire and forget)
  • Faster but no delivery guarantee
  • Used by: DNS, DHCP, VoIP, streaming

Common Ports

20/21  - FTP (File Transfer Protocol)
22     - SSH (Secure Shell)
23     - Telnet (Unsecure remote access)
25     - SMTP (Email sending)
53     - DNS (Domain Name System)
80     - HTTP (Web traffic)
443    - HTTPS (Secure web traffic)
3306   - MySQL Database
3389   - RDP (Remote Desktop Protocol)
5432   - PostgreSQL Database
8080   - HTTP Alternate

DNS (Domain Name System)

DNS translates domain names to IP addresses:

# Query DNS records
nslookup example.com

# Specific record types
dig example.com A      # IPv4 address
dig example.com MX     # Mail servers
dig example.com NS     # Name servers
dig example.com TXT    # Text records

import socket

def port_scan(target, port):

Command Line Mastery

The command line is your primary interface for security work. GUI tools are limited; real power is in the terminal.

Essential Linux Commands

File Operations

# Navigate directories

cd /path/to/dir       # Change directory

# List files

ls -lh                # Human-readable sizes

# Create/Delete

Bash Scripting Basics

#!/bin/bash
# Simple port scanner script

TARGET="192.168.1.1"
PORTS="22 80 443 3306"

echo "Scanning $TARGET..."

for port in $PORTS; do
    timeout 1 bash -c "echo >/dev/tcp/$TARGET/$port" 2>/dev/null
    if [ $? -eq 0 ]; then
        echo "Port $port is OPEN"
    else
        echo "Port $port is CLOSED"
    fi
done

echo "Scan complete!"

Hardware Basics

Understanding hardware helps you understand how attacks work at the physical level.

Key Components

CPU (Central Processing Unit)

  • Executes instructions
  • Clock speed measured in GHz
  • Multiple cores for parallel processing
  • Cache memory for fast access

RAM (Random Access Memory)

  • Temporary storage for running programs
  • Volatile (data lost when powered off)
  • Faster than disk but smaller
  • Common types: DDR4, DDR5

Storage

  • HDD: Mechanical, slower, cheaper, larger capacity
  • SSD: Electronic, faster, expensive, smaller capacity
  • NVMe: Fastest, uses PCIe interface

How Computers Execute Code

  1. Program loaded from disk to RAM
  2. CPU fetches instructions from RAM
  3. CPU decodes instructions
  4. CPU executes instructions
  5. Results stored back in RAM
  6. Data written to disk if needed

Security Relevance: Buffer overflow attacks exploit how memory is managed. Understanding the stack and heap is crucial for exploit development.
