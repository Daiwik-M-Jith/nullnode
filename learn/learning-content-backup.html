01 ‚Äî Fundamentals

We start with two core ideas: what computers are, and how to think like one. Learning to break a system starts by understanding how it normally behaves.

Quick idea: Everything in a computer is data or instructions. Programs ask the OS to run things and the OS manages resources.

Small, approachable examples

  • Files and directories are just data containers with names and properties.
  • Permissions tell the OS "who can do what" (read, write, execute).
  • Processes are running programs ‚Äî they need memory, cpu, and sometimes network access.

Try this ‚Äî list files & view a message

# Tip: click Try to run this in the simulated terminal
ls
cat welcome.txt

Try typing ls to see directory contents ‚Äî then cat welcome.txt to see welcome text.

02 ‚Äî Linux Essentials

Linux is everywhere in security. You don't need to memorize everything ‚Äî learn patterns and the commands you rely on most.

Real, practical commands

# get current directory, view hidden files, and view file contents
pwd
ls -la

Explanation: pwd prints the working directory, ls -la shows hidden files and permission bits.

Permissions example

# see sample permission line from ls -la
-rw-r--r-- 1 user user 123 Jan 1 00:00 secrets.txt

# prediction: who can read the file?
# (answer: owner and group and others can read but only owner can write)

Try ls -la and inspect the output to understand the flags on left.

03 ‚Äî Networking Basics

Everything we do in security involves connections ‚Äî either between you and a target, or between services inside a system.

Example commands (simulated)

# check local network interfaces and ping an external host
ifconfig
ping example.com

These are safe, simulated outputs ‚Äî the terminal will show realistic-looking responses which help you recognise the content you'll see on real systems.

Tip

Learn to map commands to real-world meaning: if you see an IP and a MAC address, you know which device the packet came from.

04 ‚Äî Programming & Scripting

Simple automation is your best friend ‚Äî repeatable steps let you move faster and make fewer mistakes.

Python quick example

# print from python
python -c "print('Hello from Python')"

# run a simulated file
python example.py

These are simulated but teach exact command shapes you'll run on real machines.

Automation idea

Write a small script to test many hosts: this checks port shapes and gives you data you can analyze later.

05 ‚Äî Web Application Security

Web apps speak HTTP. Learn how requests/parameters are passed and how input can be misused.

# pretend request text output from curl
curl http://example.com

# test a simple header
curl -I http://example.com

Reading responses and headers helps you identify misconfigurations and information leaks.

06 ‚Äî Exploitation & Post-Exploitation

Understand the steps: find a vulnerability, craft a probe, test on a safe lab, then validate impact.

# port discovery with nmap (simulated)
nmap 192.168.1.1

# show processes and tops
ps
top

On labs: always start with recon (nmap), then enumerate services to find potential entry points.

07 ‚Äî Reverse Engineering

Reverse engineering reads programs and deciphers how they work; the goal is to understand the shape of code and the data it handles.

# the terminal will not run real binary analysis but will show sample outputs
cat readme_for_binary.txt
man file
