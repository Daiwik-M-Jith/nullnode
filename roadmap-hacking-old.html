01

The Absolute Fundamentals ‚Äî Understanding Computers

üè† What IS a Computer Anyway?

Metaphor: Think of a computer like a small town with different buildings and workers.

  • The CPU (Brain/Mayor): The Central Processing Unit is like the mayor who makes all decisions and coordinates everything. When you click something, the CPU decides what to do next. It's FAST ‚Äî billions of decisions per second!
  • RAM (Short-term Memory/Desk): Random Access Memory is like your work desk ‚Äî it holds things you're actively using RIGHT NOW. Close a program? That stuff gets cleared off the desk. Turn off the computer? The desk is wiped clean.
  • Storage/Hard Drive (Long-term Memory/Filing Cabinet): This is where files STAY even when the computer is off ‚Äî like a filing cabinet. Your photos, documents, programs all live here permanently (until you delete them).
  • The Operating System (Town Manager/Government): Windows, macOS, Linux ‚Äî these are the "governments" that manage the town. They decide which programs can run, who gets access to what files, and keep everything organized.

Real-World Example: When you open a photo:

  1. You double-click the file (stored in the filing cabinet/hard drive)
  2. The OS (town manager) sees your request and tells the CPU (mayor)
  3. The CPU loads the photo into RAM (puts it on your desk)
  4. Your monitor displays it (you can see what's on your desk)
  5. Close the photo? It's removed from RAM but stays in storage

üìÅ Files & Folders ‚Äî The Digital Filing System

Metaphor: Files are like individual documents. Folders are like manila folders or binders that hold related documents together.

  • File Names: Like labels on documents. report.pdf tells you it's a report in PDF format. The part after the dot (`.pdf`) is called the file extension ‚Äî it tells the computer what TYPE of file it is.
  • File Paths: The address to reach a file. Like a street address: C:\Users\YourName\Documents\report.pdf means "Start at the C drive, go into Users folder, then YourName folder, then Documents folder, and there's the report."
  • Hidden Files: Some files are invisible by default (like secret passages). They start with a dot in Linux (`.secret`) or have a hidden attribute in Windows. The OS hides them so beginners don't accidentally delete important stuff.

Security Connection: Hackers look for sensitive files left in obvious places ‚Äî passwords in text files, backup files with `.bak` extension, configuration files with default passwords. Knowing how files work helps you find (and protect) secrets.

üîê Permissions ‚Äî Who Can Do What?

Metaphor: Permissions are like keys to different rooms. Some people can only LOOK in a room (read), some can CHANGE things (write), some can RUN programs (execute).

  • Read (R): You can open and view the file, but not change it. Like reading a book from the library.
  • Write (W): You can modify the file ‚Äî add, delete, or change content. Like writing in your own notebook.
  • Execute (X): You can RUN the file as a program. Like pressing "play" on a video or running an app.

Example in Linux: -rwxr-xr-- looks scary, but here's what it means:

  • First character: - means it's a file (d would mean directory/folder)
  • Next 3 characters rwx: The OWNER can read, write, and execute
  • Next 3 characters r-x: The GROUP can read and execute, but NOT write
  • Last 3 characters r--: OTHERS (everyone else) can only read

Why This Matters for Hacking: Weak permissions are a TOP security issue! If a sensitive file has -rwxrwxrwx (read/write/execute for EVERYONE), any user can steal or modify it. Hackers look for these mistakes!

‚öôÔ∏è Processes ‚Äî Programs in Action

Metaphor: A process is like a worker actively doing a job. The program file is the job description, the process is the person actually working.

  • Starting a Process: When you double-click an app, the OS creates a process ‚Äî loads the program into RAM and starts executing instructions.
  • Process ID (PID): Every process gets a unique number (like an employee ID). The OS uses this to track and manage it.
  • Killing a Process: If a program freezes, you can "kill" its process (fire the worker). In Windows: Task Manager. In Linux: kill <PID>

Security Connection: Hackers can inject malicious code into running processes, hide malware as fake system processes, or exploit vulnerable processes to gain control. Understanding processes helps you spot what's NORMAL vs SUSPICIOUS.

üåê Networking Basics ‚Äî Computers Talking to Each Other

Metaphor: The internet is like a massive postal system. Computers send "letters" (data packets) to each other using addresses.

  • IP Address: Like a house address. `192.168.1.1` is a typical home network address. `8.8.8.8` is Google's public DNS server. Every device on the internet has one.
  • Ports: Like apartment numbers. One IP address (building) can host many services (apartments). Port 80 = web traffic (HTTP), Port 443 = secure web (HTTPS), Port 22 = SSH (remote login).
  • Packets: Data is broken into small chunks called packets (like breaking a long letter into pages). Each packet has a "from" address, "to" address, and the actual data inside.

Real Example: When you visit google.com:

  1. Your computer asks a DNS server "What's the IP address for google.com?" (like looking up a phone number)
  2. DNS replies: "It's 142.250.80.46"
  3. Your computer sends a packet to 142.250.80.46 on Port 443 saying "Hey, send me your homepage!"
  4. Google's server sends back packets with the HTML, CSS, images
  5. Your browser assembles the packets and displays the page

Why This Matters: Hackers intercept packets (sniffing), pretend to be other computers (spoofing), or flood servers with fake packets (DDoS attacks). Knowing how networking works is ESSENTIAL.

üìù What is "Code" / Programming?

Metaphor: Code is like a recipe. You write step-by-step instructions, and the computer follows them EXACTLY (even if your instructions are wrong!).

  • Variables: Named storage boxes. age = 25 means "create a box called 'age' and put 25 inside."
  • If Statements: Decisions. "If it's raining, bring umbrella. Otherwise, don't." Computers make millions of these tiny decisions.
  • Loops: Repetition. "Do this 100 times" or "Keep doing this until X happens."
  • Functions: Reusable recipes. Instead of writing "make a sandwich" steps every time, define it once and call "makeSandwich()" whenever you need it.

Simple Example in Python:

age = 16
if age >= 18:
    print("You can vote!")
else:
    print("Too young to vote.")

This checks if age is 18 or more. If yes, print one message. If no, print another.

Why Hackers Need Code: Automation! Instead of manually checking 1000 servers, write a script to do it in seconds. Parsing logs, crafting exploits, building tools ‚Äî all require coding.

üíª The Command Line / Terminal ‚Äî The Power User Interface

Metaphor: The command line is like texting directly with the computer. Instead of clicking icons (pointing at things), you TYPE exactly what you want.

  • Why Use It?: Faster, more powerful, scriptable. Many hacking tools ONLY have command-line interfaces.
  • Basic Commands (Linux/Mac):
    • pwd = Print Working Directory (where am I?)
    • ls = List files in current folder
    • cd <folder> = Change Directory (move into a folder)
    • cat <file> = Show file contents
    • mkdir <name> = Make a new directory

Real Example: You want to see what's in your Documents folder:

cd ~/Documents    # Move into Documents folder
ls                # List all files there
cat report.txt    # Read report.txt

Why This Matters: Ethical hackers spend 80% of their time in terminals. You'll use command-line tools like nmap, metasploit, burpsuite, etc. Get comfortable NOW.

Fundamentals II ‚Äî Going Deeper Under the Hood

Purpose: Chapter 1 taught you WHAT things are. Chapter 2 teaches you HOW they work internally. This bridges the gap between "user" and "power user."

üîß Operating System Internals ‚Äî What Happens When You Click?

Metaphor: The OS is like a master juggler managing thousands of balls (programs) at once. Let's see HOW it does this magic.

Processes vs Threads

  • Process: A running program with its own memory space. Like a restaurant (isolated, has its own kitchen/dining area).
  • Thread: A smaller unit within a process. Like waiters in the restaurant ‚Äî they all share the same kitchen but work independently.
  • Why This Matters: A web browser is ONE process with MANY threads (one per tab). If one tab crashes, others survive. Hackers exploit thread synchronization bugs!

Memory Management ‚Äî Where Does Data Live?

Metaphor: Imagine memory as a hotel with rooms numbered 0x00000000 to 0xFFFFFFFF (hexadecimal addresses).

  • Stack: Fast, automatic memory for function calls and local variables. Like sticky notes ‚Äî quick to write, quick to discard. Limited size (usually 1-8MB).
  • Heap: Manually managed memory for dynamic data. Like storage units you rent ‚Äî you must explicitly free them when done or cause a "memory leak."
  • Buffer Overflow Attack: If a program writes MORE data than a buffer can hold, it spills into adjacent memory ‚Äî potentially overwriting critical data or injecting malicious code!

Real Example of Buffer Overflow:

char password[8];  // 8-byte buffer
gets(password);    // Reads user input (UNSAFE!)
// User types "AAAAAAAAAAAAAAAAAAAA" (20 A's)
// Bytes 1-8 fill password buffer
// Bytes 9-20 OVERFLOW into adjacent memory!
// Could overwrite return address ‚Üí attacker gains control!

System Calls ‚Äî Asking the OS for Help

Programs can't directly access hardware (security risk!). They must ASK the OS through system calls.

  • open() ‚Äî "OS, please open this file for me"
  • read() ‚Äî "Give me data from this file"
  • write() ‚Äî "Let me write data here"
  • execve() ‚Äî "Launch this program"

Security Implication: Hackers exploit bugs in how programs make system calls ‚Äî passing malicious parameters to trick the OS.

üåê Advanced Networking ‚Äî How the Internet REALLY Works

The OSI Model ‚Äî 7 Layers of Communication

Metaphor: Sending a package through multiple checkpoints, each adding a label/wrapper.

  1. Layer 7 (Application): The actual message ‚Äî HTTP, DNS, email. "Hey Google, send me your homepage!"
  2. Layer 4 (Transport): TCP/UDP ‚Äî breaks message into packets, adds port numbers. "This goes to port 443."
  3. Layer 3 (Network): IP addresses. "From 192.168.1.5 to 8.8.8.8."
  4. Layer 2 (Data Link): MAC addresses (physical hardware addresses). "From my network card to the router."
  5. Layer 1 (Physical): Actual electrical signals/light pulses on wires/fiber.

Why Hackers Care: Each layer can be attacked! Layer 2 (ARP spoofing), Layer 3 (IP spoofing), Layer 7 (SQL injection). Knowing the layers helps you understand WHERE an attack happens.

DNS ‚Äî The Internet's Phone Book

Metaphor: DNS translates human-friendly names (google.com) into computer-friendly numbers (142.250.80.46).

How It Works:

  1. You type "facebook.com" in browser
  2. Browser asks local DNS resolver: "What's the IP for facebook.com?"
  3. If cached ‚Üí instant answer. If not ‚Üí resolver asks root DNS servers ‚Üí .com servers ‚Üí facebook.com's authoritative servers
  4. Answer returned: "157.240.241.35"
  5. Browser connects to that IP

DNS Attacks: DNS spoofing (fake DNS responses), DNS tunneling (hiding data in DNS queries), typosquatting (gooogle.com ‚Üí malicious site).

NAT ‚Äî Why 192.168.x.x is EVERYWHERE

Problem: Only ~4 billion IPv4 addresses exist. Billions of devices need internet!

Solution: Network Address Translation (NAT). Your home router has ONE public IP (visible to internet) but gives PRIVATE IPs (192.168.x.x) to all devices inside.

How It Works:

  • Your laptop (192.168.1.10) sends packet to google.com
  • Router replaces source IP with its public IP (203.0.113.5)
  • Google replies to 203.0.113.5
  • Router sees reply, translates back to 192.168.1.10, forwards to your laptop

Security Implication: NAT acts as a firewall ‚Äî external attackers can't directly reach your 192.168.x.x devices. BUT port forwarding/UPnP can expose them!

üíª Scripting Mastery ‚Äî Automation is Power

Why Scripting is Essential for Hackers

Manual work = slow. Scripts = fast, repeatable, scalable.

  • Recon Automation: Scan 1000 IPs for open ports in minutes
  • Log Parsing: Find suspicious patterns in gigabytes of logs
  • Exploit Development: Test payloads with slight variations automatically
  • Report Generation: Turn raw data into clean PDF reports

Bash/Shell Scripting ‚Äî The Linux Power Tool

Example Script: Find all files modified in last 24 hours

#!/bin/bash
# Save as: find_recent.sh

DIRECTORY="/var/log"
TIME="24h"

echo "Files modified in last $TIME in $DIRECTORY:"
find "$DIRECTORY" -type f -mtime -1 -ls

Python Scripting ‚Äî The Swiss Army Knife

import requests
import sys

# Simple port scanner
def scan_port(ip, port):
    try:
        sock = socket.socket()
        sock.settimeout(0.5)
        result = sock.connect_ex((ip, port))
        sock.close()
        return result == 0  # True if open
    except:
        return False

target = sys.argv[1]
for port in [21, 22, 80, 443, 3306, 8080]:
    if scan_port(target, port):
        print(f"Port {port} is OPEN")
    else:
        print(f"Port {port} is closed")

üêô Version Control with Git ‚Äî Track EVERYTHING

Metaphor: Git is like a time machine for your code. Every change is saved, you can go back to any point.

  • Repository (Repo): A project folder tracked by Git
  • Commit: A snapshot of your files at a specific time. Like saving your game.
  • Branch: A parallel timeline. Experiment without affecting the main code.
  • Remote: Online copy (GitHub/GitLab) ‚Äî backup + collaboration

Basic Git Workflow:

git init               # Start tracking this folder
git add script.py      # Stage file for commit
git commit -m "Added port scanner"  # Save snapshot
git push origin main   # Upload to GitHub

Why Hackers Use Git: Version control for exploit code, collaborate on tools, maintain public PoCs (Proof of Concepts), track vulnerability research.

üêõ Debugging & Troubleshooting ‚Äî Finding the "Why"

Using strace ‚Äî See What a Program is Doing

What it does: Shows EVERY system call a program makes.

strace ls
# Output shows:
# open("/etc/passwd", O_RDONLY) = 3
# read(3, "root:x:0:0:root:/root:/bin/bash\n...", 4096) = 2314
# ...

You see EXACTLY which files it opens, what it reads, what system calls it makes. Invaluable for understanding malware!

GDB ‚Äî The Debugger

Step through programs line-by-line, inspect memory, set breakpoints.

gdb ./program
(gdb) break main     # Pause at main function
(gdb) run            # Start program
(gdb) print var      # Show variable value
(gdb) continue       # Keep going

üê≥ Docker Basics ‚Äî Reproducible Environments

Problem: "It works on my machine!" ‚Äî software behaves differently on different systems.

Solution: Docker containers package software + dependencies into a portable unit.

Metaphor: Containers are like shipping containers ‚Äî standardized boxes that work anywhere.

  • Image: A blueprint/recipe (like a class in programming)
  • Container: A running instance (like an object)
  • Dockerfile: Instructions to build an image

Example Dockerfile:

FROM ubuntu:22.04
RUN apt-get update && apt-get install -y python3 nmap
COPY scan.py /app/
CMD ["python3", "/app/scan.py"]

Why Use Docker for Hacking? Isolate malware analysis, quickly spin up vulnerable test environments, share reproducible exploit setups.
