1. Memory Management: Under the Hood

When you create a variable like x = 10, where does it go? Understanding memory is what separates "coders" from "software engineers".

Stack vs. Heap

The Stack (Ordered & Fast)

Think of a stack of plates. You can only add to the top and remove from the top.

  • What goes here: Local variables, function calls.
  • Speed: Extremely fast access.
  • Management: Automatic. When a function ends, its data is "popped" off.
  • Limit: Small size. If you use too much (infinite recursion), you get a Stack Overflow.

The Heap (Chaotic & Large)

Think of a giant messy room where you can throw things anywhere.

  • What goes here: Objects, large data structures, global variables.
  • Speed: Slower. You need a "pointer" (address) to find your data.
  • Management: Manual (in C/C++) or Garbage Collected (in Python/Java).
  • Limit: Limited only by your physical RAM.

2. Data Structures: Organizing Chaos

A Data Structure is just a way to store data so you can use it efficiently. Choosing the right one can make your code 100x faster.

Arrays (Lists)

Concept: Items stored side-by-side in memory.
Good for: Accessing items by index (list[5] is instant).
Bad for: Inserting/deleting items in the middle (everything else has to shift).

Linked Lists

Concept: A chain where each item holds the data and a "pointer" to the next item.
Good for: Inserting/deleting items (just change the pointer).
Bad for: Finding the 100th item (you have to walk the whole chain).

Hash Maps (Dictionaries)

Concept: Key-Value pairs. Like a real dictionary.
Good for: Finding data instantly by key (user["name"]).
The Magic: It uses a "hash function" to turn the key into a memory address directly.

3. Algorithms & Big O Notation

An Algorithm is a step-by-step recipe to solve a problem. Big O Notation is how we measure how "fast" or "slow" an algorithm is as the data grows.

O(1) - Constant Time (The Best)

Takes the same time regardless of data size.
Example: Accessing an array index.

O(n) - Linear Time (Okay)

Time grows directly with data size. 10 items = 10 seconds. 100 items = 100 seconds.
Example: Looping through a list to find a value.

O(n¬≤) - Quadratic Time (Bad)

Time grows exponentially. A loop inside a loop.
Example: Comparing every item in a list with every other item.
