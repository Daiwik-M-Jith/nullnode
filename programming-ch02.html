Chapter 02

Control Flow

Make your programs smart. Control flow lets your code make decisions and repeat actions - the two things that make programs actually useful.

Up until now, your programs run from top to bottom, line by line, every time. But real programs need to make decisions: "If the password is correct, let them in. Otherwise, show an error."

They also need to repeat actions: "Keep asking for input until they type 'quit'" or "Process every item in this list."

Control flow is what separates a simple calculator from a smart assistant. It's the difference between a dumb script and an intelligent program. Master these concepts and you can build almost anything.

üö¶ Traffic Lights of Code

Think of control flow like traffic lights controlling the flow of cars:

  • If statements (red/green): Stop or go based on conditions
  • Loops (roundabouts): Keep going in circles until your exit appears
  • Break (turn off): Emergency exit from the roundabout

Without traffic lights, cars would crash. Without control flow, programs would be useless - just doing the same thing every time regardless of circumstances.

üîÄ Conditionals

Make decisions based on conditions. Different inputs = different behavior.

  • if statements
  • else and elif
  • Comparison operators
  • Logical operators (and, or, not)

üîÑ Loops

Repeat actions multiple times. Process collections item by item.

  • while loops (repeat while true)
  • for loops (iterate over items)
  • range() function
  • break and continue

The Power of Control Flow

With variables + control flow, you can build almost anything. Games, apps, automation scripts, websites - they're all just decisions and loops. The Tic-Tac-Toe game at the end of this chapter uses only concepts from Ch01 and Ch02!

Conditional Statements: Making Decisions

The if statement is the most important control structure. It lets your code take different paths based on conditions. Without conditionals, every run of your program would be identical - boring and useless.

üõ§Ô∏è The Crossroads Metaphor

Imagine you're walking and reach a fork in the road. You check your map (the condition):

  • If the destination is north, go left
  • Otherwise (else), go right

That's exactly what if statements do - they check a condition and choose which code path to follow.

The Anatomy of an If Statement

Comparison Operators: The Foundation of Conditions

Conditions use comparisons. Every if statement boils down to asking a yes/no question. Comparison operators are how you ask those questions. They compare two values and return True or False.

‚öñÔ∏è The Scales of Justice

Think of comparison operators like scales - you put something on each side and they tell you how they relate:

  • == Are both sides equal? (balanced)
  • != Are the sides different? (unbalanced)
  • > < Which side is heavier? (which is greater)

The scales always give you a yes (True) or no (False) answer - never maybe!

The Six Comparison Operators

# ==================== ALL COMPARISON OPERATORS ==================== x = 10 x == 10 # True Equal to (is x exactly 10?) x != 5 # True Not equal to (is x different from 5?) x > 5 # True Greater than (is x more than 5?) x < 5 # False Less than (is x less than 5?) x >= 10 # True Greater than OR equal to (is x at least 10?) x <= 10 # True Less than OR equal to (is x at most 10?) # ==================== REMEMBER: == vs = ==================== # = is ASSIGNMENT (storing a value) x = 5 # Put 5 into x # == is COMPARISON (asking a question) x == 5 # Is x equal to 5? Returns True # This is THE most common mistake: # if x = 5: # ‚ùå SyntaxError! You're assigning, not comparing # if x == 5: # ‚úì Correct comparison # ==================== USING IN CONDITIONS ==================== age = 25 if age >= 18: print("Adult") if age != 0: print("Age is set") if age == 25: print("Exactly 25") # ==================== CHAINING COMPARISONS ==================== # Python lets you chain comparisons (unique feature!) score = 75 # Instead of: if score >= 70 and score < 80: print("C grade") # You can write: if 70 <= score < 80: print("C grade") # This reads more naturally: "Is 70 less-than-or-equal score less-than 80?" # More examples: x = 5 1 < x < 10 # True (x is between 1 and 10) 1 < x < 3 # False (x is not between 1 and 3) 10 >= x >= 1 # True (x is between 1 and 10, inclusive) # You can chain as many as you want: a, b, c, d = 1, 2, 3, 4 a < b < c < d # True (they're in ascending order)

Comparing Different Types

# ==================== COMPARING STRINGS ==================== name = "Alice" name == "Alice" # True (exact match) name == "alice" # False (CASE SENSITIVE!) name == "ALICE" # False (case matters) name.lower() == "alice" # True (convert to lowercase first) # String comparisons are lexicographic (dictionary order): "apple" < "banana" # True (a comes before b) "apple" < "apricot" # True (compare character by character) "Apple" < "apple" # True (uppercase letters come before lowercase!) # Why? It uses ASCII/Unicode values: # 'A' = 65, 'Z' = 90 # 'a' = 97, 'z' = 122 # So all uppercase < all lowercase "10" < "9" # True! (string comparison, not numeric) # "1" comes before "9" in first character # ==================== COMPARING NUMBERS ==================== 5 == 5.0 # True (int and float can be equal) 10 > 3 # True -5 < 0 # True 3.14159 >= 3.14 # True # Be careful with floats! 0.1 + 0.2 == 0.3 # False! (floating point precision) # 0.1 + 0.2 is actually 0.30000000000000004 # For floats, use approximate comparison: import math math.isclose(0.1 + 0.2, 0.3) # True # ==================== TYPES THAT CAN'T BE COMPARED ==================== # Comparing incompatible types causes TypeError: # 5 < "5" # ‚ùå TypeError: '<' not supported between int and str # [1, 2] < "hello" # ‚ùå TypeError # == works on different types but returns False: 5 == "5" # False (different types = not equal) True == 1 # True (special case: True is 1, False is 0) False == 0 # True

Checking Types and Existence

# ==================== CHECKING IF VALUE EXISTS ==================== # Check if variable has a "real" value (not empty/zero): name = "Alice" if name: # True because non-empty string print("Name exists") count = 0 if count: # False because 0 is falsy print("Count exists") # This won't print! # Better for numbers: if count is not None: # Check if it's actually set print(f"Count is {count}") # ==================== CHECKING DATA TYPES ==================== value = "42" # Using type(): if type(value) == str: print("It's a string") if type(value) == int: print("It's an integer") # Won't print # Better way - isinstance(): if isinstance(value, str): print("It's a string") if isinstance(value, (int, float)): # Check multiple types print("It's a number") # ==================== MEMBERSHIP TESTING ==================== # 'in' operator checks if something is inside something else: "a" in "apple" # True "x" in "apple" # False 3 in [1, 2, 3, 4, 5] # True "key" in {"key": "val"} # True (checks keys in dictionaries) # Use with if statements: letter = "e" vowels = "aeiou" if letter in vowels: print(f"{letter} is a vowel") else: print(f"{letter} is a consonant")

üí° Comparison Best Practices

  • Always use == for comparison, never =
  • For None, use is None instead of == None
  • For string comparison, decide if case matters and use .lower() if needed
  • Avoid comparing floats with ==; use math.isclose()
  • Use chained comparisons for ranges: 0 <= x < 100

Logical Operators: Combining Conditions

Sometimes you need to check multiple conditions at once. Logical operators let you combine conditions into more complex expressions. This is how you build sophisticated decision-making into your programs.

üö™ The Nightclub Bouncer

Imagine a bouncer checking entry requirements:

  • AND: "You need to be 21+ AND have valid ID" (both required)
  • OR: "VIP pass OR on the guest list gets you in" (either works)
  • NOT: "NOT on the banned list" (reverses the check)

Logical operators work exactly the same way - combining conditions to make a final yes/no decision.

The Three Logical Operators

AND - Both Must Be True

Returns True only if ALL conditions are True. If any is False, the result is False.

age = 25 has_license = True if age >= 18 and has_license: print("You can drive") # age >= 18? True ‚úì # has_license? True ‚úì # Both True ‚Üí Result: True

OR - At Least One True

Returns True if ANY condition is True. Only False if ALL are False.

is_weekend = True is_holiday = False if is_weekend or is_holiday: print("No work today!") # is_weekend? True ‚úì # is_holiday? False ‚úó # At least one True ‚Üí Result: True

NOT - Reverses the Value

Flips True to False and False to True. Inverts any condition.

is_raining = False if not is_raining: print("Go outside!") # is_raining? False # not False ‚Üí True # Condition is True ‚Üí Code runs

Truth Tables - How They Work

# ==================== AND TRUTH TABLE ==================== # Both must be True to get True True and True # True ‚úì + ‚úì = ‚úì True and False # False ‚úì + ‚úó = ‚úó False and True # False ‚úó + ‚úì = ‚úó False and False # False ‚úó + ‚úó = ‚úó # Think of it as: "Do I have EVERYTHING I need?" # ==================== OR TRUTH TABLE ==================== # Only need one True to get True True or True # True (both work) True or False # True (first works) False or True # True (second works) False or False # False (neither works) # Think of it as: "Do I have AT LEAST ONE thing I need?" # ==================== NOT TRUTH TABLE ==================== # Simply flips the value not True # False not False # True # Think of it as: "The opposite" # ==================== COMBINING THEM ==================== # You can chain multiple logical operators: a = True b = False c = True a and b and c # False (b is False) a or b or c # True (a is True) a and (b or c) # True (b or c is True, a is True) not (a and b) # True (a and b is False, not makes it True)

Order of Operations (Precedence)

# ==================== OPERATOR PRECEDENCE ==================== # Order (highest to lowest): # 1. Parentheses () # 2. Comparison operators (==, !=, <, >, <=, >=) # 3. not # 4. and # 5. or # This means: a = True b = False c = True # This: a or b and c # Is evaluated as: a or (b and c) # 'and' happens first! # = True or (False and True) # = True or False # = True # NOT as: (a or b) and c # This would also be True, but for different reasons # ==================== USE PARENTHESES FOR CLARITY ==================== # Even if you know the precedence, parentheses make code clearer: age = 25 has_id = True is_vip = False # Hard to read: if age >= 21 and has_id or is_vip: print("Enter") # Much clearer: if (age >= 21 and has_id) or is_vip: print("Enter") # Or even better, use intermediate variables: meets_requirements = age >= 21 and has_id if meets_requirements or is_vip: print("Enter")

Short-Circuit Evaluation

# ==================== WHAT IS SHORT-CIRCUITING? ==================== # Python is "lazy" - it stops evaluating as soon as the result is known # With AND: # If the first condition is False, the whole thing is False # So Python doesn't even look at the second condition! False and (1/0) # Returns False (doesn't crash!) # 1/0 would cause ZeroDivisionError, but Python never evaluates it # With OR: # If the first condition is True, the whole thing is True # So Python doesn't look at the second condition! True or (1/0) # Returns True (doesn't crash!) # ==================== PRACTICAL USES ==================== # 1. Safe checking (avoid errors): name = "" # Empty string # This would crash if name is empty: # if name[0] == "A": # This is safe: if name and name[0] == "A": print("Name starts with A") # If name is empty (falsy), Python short-circuits and never tries name[0] # 2. Default values: user_input = "" username = user_input or "Guest" # "Guest" (because user_input is falsy) user_input = "Alice" username = user_input or "Guest" # "Alice" # 3. Safe division: denominator = 0 if denominator != 0 and (100 / denominator) > 10: print("Big result") # Won't crash because first condition is False ‚Üí short-circuits

Practical Examples

‚ö†Ô∏è Common Logical Operator Mistakes

  • Using 'and' when you mean 'or': "age is 18 or 21" should be age == 18 or age == 21
  • Forgetting to repeat the variable: if age == 18 or 21 ‚ùå ‚Üí if age == 18 or age == 21 ‚úì
  • Double negatives: if not (not is_valid) ‚Üí just use if is_valid
  • Overcomplicating: if condition == True ‚Üí just use if condition

Multiple Conditions with elif

What if you have more than two possible paths? Use elif (else if) to check multiple conditions in sequence. This is how you handle situations with many possible outcomes.

üé∞ The Sorting Machine

Imagine a machine that sorts packages by size:

  • If weight > 50kg ‚Üí Heavy bin
  • Else if weight > 20kg ‚Üí Medium bin
  • Else if weight > 5kg ‚Üí Light bin
  • Else ‚Üí Tiny bin

The machine checks each condition in order and sends the package to the FIRST matching bin. Once matched, it doesn't check the rest.

The elif Structure

# ==================== BASIC STRUCTURE ==================== # if condition1: # code for condition1 # elif condition2: # code for condition2 # elif condition3: # code for condition3 # else: # code if nothing matched # ==================== GRADING SYSTEM EXAMPLE ==================== score = int(input("Enter score (0-100): ")) if score >= 90: grade = "A" comment = "Excellent!" elif score >= 80: grade = "B" comment = "Good job!" elif score >= 70: grade = "C" comment = "Satisfactory" elif score >= 60: grade = "D" comment = "Needs improvement" else: grade = "F" comment = "Failed - see me after class" print(f"Score: {score}") print(f"Grade: {grade}") print(f"Comment: {comment}") # ==================== HOW IT WORKS ==================== # For score = 85: # 1. Is 85 >= 90? No ‚Üí skip # 2. Is 85 >= 80? Yes! ‚Üí grade = "B", stop checking # 3. Never reaches score >= 70, 60, or else # Only ONE block ever runs! # ==================== THE FLOW ==================== # [score >= 90?] # | # No ‚Üì Yes ‚Üí "A" # [score >= 80?] # | # No ‚Üì Yes ‚Üí "B" # [score >= 70?] # | # No ‚Üì Yes ‚Üí "C" # [score >= 60?] # | # No ‚Üì Yes ‚Üí "D" # ‚Üì # "F"

‚ö†Ô∏è Order Matters Critically!

Python checks conditions from top to bottom and runs the FIRST matching block.

# WRONG ORDER - Everyone gets D! score = 95 if score >= 60: # 95 >= 60? Yes! Gets D grade = "D" elif score >= 70: # Never reached grade = "C" elif score >= 80: # Never reached grade = "B" elif score >= 90: # Never reached grade = "A" # Always order from most restrictive to least!

elif vs Multiple ifs

# ==================== ELIF - MUTUALLY EXCLUSIVE ==================== # Use elif when only ONE condition should match: age = 25 if age < 13: category = "child" elif age < 20: category = "teenager" elif age < 65: category = "adult" else: category = "senior" print(category) # "adult" # Only ONE category is printed # ==================== MULTIPLE IFS - INDEPENDENT CHECKS ==================== # Use separate ifs when conditions are independent: score = 85 achievements = [] if score >= 50: achievements.append("Passed") if score >= 70: achievements.append("Good student") if score >= 90: achievements.append("Honor roll") print(achievements) # ['Passed', 'Good student'] # Multiple can be true! # ==================== THE KEY DIFFERENCE ==================== # elif: "Check this only if previous conditions were False" # if: "Check this regardless of previous conditions" # Wrong use of elif when you want all matches: score = 95 if score >= 50: print("Passed") elif score >= 70: # This won't print! print("Good student") # Because first condition was True elif score >= 90: # This won't print either! print("Honor roll") # Correct - use separate ifs: score = 95 if score >= 50: print("Passed") # Prints if score >= 70: print("Good student") # Prints if score >= 90: print("Honor roll") # Prints

üí° When to Use What

  • if-else: Two possible paths
  • if-elif-else: Multiple mutually exclusive paths
  • Multiple ifs: Independent conditions that can all be true
  • Ternary: Simple value assignment based on condition

Nested Conditions

You can put if statements inside other if statements. This is called nesting. It's useful when decisions depend on previous decisions, but too much nesting makes code hard to read.

üè¢ The Security Checkpoint

Imagine entering a secure building:

  • First check: Do you have an ID badge?
  • If yes: Is it still valid (not expired)?
  • If yes: Do you have clearance for this floor?

Each check only happens if the previous check passed. That's nesting in action.

Basic Nested If

# ==================== NESTED IF STRUCTURE ==================== age = int(input("Enter age: ")) has_id = input("Do you have ID? (yes/no): ").lower() == "yes" if age >= 18: # Only check ID if they're old enough if has_id: print("‚úì Welcome to the club!") else: print("‚úó Adult but no ID - cannot enter") else: print("‚úó Too young to enter") # ==================== HOW IT FLOWS ==================== # [age >= 18?] # / \ # Yes No # | | # [has_id?] "Too young" # / \ # Yes No # | | # "Welcome" "No ID" # ==================== MULTIPLE LEVELS ==================== user_type = "admin" is_logged_in = True has_2fa = True if is_logged_in: print("You are logged in") if user_type == "admin": print("You have admin privileges") if has_2fa: print("Full access granted") else: print("Please enable 2FA for full access") else: print("Standard user access") else: print("Please log in first")

When Nesting is Appropriate

# ==================== GOOD USE: DEPENDENT CONDITIONS ==================== # Checking a file before processing it import os filename = "data.txt" if os.path.exists(filename): print(f"File {filename} exists") if os.path.isfile(filename): print("It's a file (not a directory)") # Now safe to read the file with open(filename) as f: content = f.read() if content: print(f"File has {len(content)} characters") else: print("File is empty") else: print("It's a directory, not a file") else: print(f"File {filename} does not exist") # ==================== GOOD USE: VALIDATION CHAIN ==================== # Form validation - each check depends on previous passing email = input("Email: ") password = input("Password: ") confirm = input("Confirm password: ") if email: # Not empty if "@" in email and "." in email: # Basic format check if password: # Not empty if len(password) >= 8: # Long enough if password == confirm: # Matches confirmation print("‚úì Account created successfully!") else: print("‚úó Passwords don't match") else: print("‚úó Password must be 8+ characters") else: print("‚úó Password is required") else: print("‚úó Invalid email format") else: print("‚úó Email is required")

Flattening Nested Conditions

# ==================== TOO MUCH NESTING (PYRAMID OF DOOM) ==================== # This is hard to read and maintain: if condition1: if condition2: if condition3: if condition4: do_something() # Deeply nested! # ==================== TECHNIQUE 1: USE AND ==================== # Instead of: if age >= 18: if has_id: print("Welcome") # Use: if age >= 18 and has_id: print("Welcome") # ==================== TECHNIQUE 2: EARLY RETURNS ==================== # Instead of deep nesting, return/exit early on failures: # (This works best in functions) def check_access(age, has_id, has_pass): if age < 18: return "Too young" if not has_id: return "No ID" if not has_pass: return "No pass" return "Welcome!" # Only reached if all checks pass # ==================== TECHNIQUE 3: GUARD CLAUSES ==================== # Check for invalid cases first, then proceed: email = "test@example.com" password = "secretpassword" # Guard clauses (check failures first): if not email: print("Email required") elif "@" not in email: print("Invalid email") elif not password: print("Password required") elif len(password) < 8: print("Password too short") else: # All validations passed - proceed with the main logic print("Creating account...") # Much flatter than nested ifs! # ==================== TECHNIQUE 4: INTERMEDIATE VARIABLES ==================== # Break complex conditions into named pieces: user = {"age": 25, "verified": True, "subscription": "premium"} # Hard to read: if user["age"] >= 18 and user["verified"] and user["subscription"] in ["premium", "enterprise"]: print("Full access") # Better with variables: is_adult = user["age"] >= 18 is_verified = user["verified"] has_premium = user["subscription"] in ["premium", "enterprise"] if is_adult and is_verified and has_premium: print("Full access")

üí° Nesting Best Practices

  • Max 3 levels: If you're nesting deeper, refactor
  • Use and: Combine simple conditions instead of nesting
  • Guard clauses: Handle error cases first, then proceed
  • Extract functions: Move nested logic into separate functions
  • Meaningful names: Use intermediate variables for complex conditions

While Loops: Repeat While True

Loops repeat code. A while loop keeps running as long as its condition is True. Use while loops when you don't know how many times you'll need to repeat - you just know when to stop.

üöø The Shower Metaphor

A while loop is like taking a shower:

  • while you're still soapy...
  • ...keep rinsing

You don't know exactly how many rinses you'll need. You just keep going until you're clean (condition becomes False). That's the essence of a while loop.

The Anatomy of a While Loop

# ==================== BASIC STRUCTURE ==================== # while condition: # code to repeat # (something should eventually make condition False!) # ==================== SIMPLE COUNTING EXAMPLE ==================== count = 1 while count <= 5: print(count) count += 1 # CRUCIAL: Update the variable! print("Done!") # Output: # 1 # 2 # 3 # 4 # 5 # Done! # ==================== STEP BY STEP ==================== # Iteration 1: count=1, 1<=5? Yes ‚Üí print 1, count=2 # Iteration 2: count=2, 2<=5? Yes ‚Üí print 2, count=3 # Iteration 3: count=3, 3<=5? Yes ‚Üí print 3, count=4 # Iteration 4: count=4, 4<=5? Yes ‚Üí print 4, count=5 # Iteration 5: count=5, 5<=5? Yes ‚Üí print 5, count=6 # Iteration 6: count=6, 6<=5? No ‚Üí EXIT LOOP # Print "Done!" # ==================== THE THREE PARTS OF A WHILE LOOP ==================== # 1. INITIALIZATION: Set up the variable(s) before the loop count = 1 # 2. CONDITION: Check if we should continue while count <= 5: print(count) # 3. UPDATE: Change something so the condition eventually becomes False count += 1 # Missing any of these three causes problems!

‚ö†Ô∏è Infinite Loop Warning!

If the condition never becomes False, the loop runs FOREVER! This freezes your program.

# INFINITE LOOP - DON'T DO THIS: count = 1 while count <= 5: print(count) # Forgot to increment! count stays 1 forever # Another infinite loop: while True: print("Forever...") # Never stops # To stop an infinite loop: Ctrl+C in terminal

Always make sure something in your loop eventually makes the condition False!

üí° While vs For: When to Use Which?

  • Use while when: You don't know how many iterations you need
  • Use while when: Looping until some condition changes
  • Use while True when: You'll break out based on complex conditions
  • Use for when: You know the number of iterations
  • Use for when: Iterating over a collection

For Loops: Iterate Over Collections

For loops are used when you know what you're iterating over - a range of numbers, characters in a string, items in a list. They're the most commonly used loop in Python because most real-world tasks involve processing collections.

üìã The Checklist Metaphor

A for loop is like going through a checklist:

  • for each item in the list...
  • ...do something with that item
  • ...then move to the next item

You don't need to track the count yourself - Python handles it automatically.

Break and Continue: Loop Control

Sometimes you need to exit a loop early or skip certain iterations. That's what break and continue are for. They give you fine-grained control over loop execution.

üèÉ The Race Metaphor

  • break: You stop running and leave the track entirely (exit the loop)
  • continue: You skip this lap but keep running (skip to next iteration)

Both are useful for different situations. Use break when you've found what you need or an error occurred. Use continue when this particular item isn't relevant.

Break - Exit Loop Immediately

# ==================== BASIC BREAK ==================== # break immediately exits the entire loop for i in range(10): if i == 5: break # Exit when i is 5 print(i) # Output: 0, 1, 2, 3, 4 (stops before printing 5) # ==================== SEARCH AND FIND ==================== # Find first even number for num in [1, 3, 5, 8, 9, 10]: if num % 2 == 0: print(f"First even number: {num}") # 8 break # Find item in list items = ["apple", "banana", "cherry", "date"] search = "cherry" for i, item in enumerate(items): if item == search: print(f"Found '{search}' at index {i}") break else: # This else belongs to the for loop! # It runs ONLY if the loop completed without break print(f"'{search}' not found") # ==================== BREAK IN NESTED LOOPS ==================== # break only exits the INNERMOST loop! for i in range(3): print(f"Outer loop i = {i}") for j in range(3): if j == 1: break # Only breaks inner loop print(f" Inner loop j = {j}") # Output: # Outer loop i = 0 # Inner loop j = 0 (j=1 breaks, inner loop restarts for i=1) # Outer loop i = 1 # Inner loop j = 0 # Outer loop i = 2 # Inner loop j = 0 # ==================== EARLY EXIT ON ERROR ==================== data = [10, 20, "error", 30] total = 0 for item in data: if not isinstance(item, (int, float)): print(f"Error: Non-numeric value '{item}' found!") break total += item print(f"Sum of valid items: {total}") # 30

Continue - Skip to Next Iteration

# ==================== BASIC CONTINUE ==================== # continue skips the rest of THIS iteration and goes to the next for i in range(10): if i % 2 == 0: continue # Skip even numbers print(i) # Output: 1, 3, 5, 7, 9 (only odd numbers) # ==================== SKIP INVALID DATA ==================== data = [10, -5, 20, -3, 15, 0, 25] total = 0 for value in data: if value <= 0: continue # Skip non-positive numbers total += value print(f"Sum of positive numbers: {total}") # 70 # ==================== SKIP WITH MESSAGE ==================== numbers = [1, 2, "three", 4, None, 6] for num in numbers: if not isinstance(num, (int, float)): print(f"Skipping invalid value: {num}") continue # This code only runs for valid numbers print(f"Processing: {num}, squared = {num ** 2}") # ==================== FILTERING PATTERN ==================== # Process only items meeting criteria users = [ {"name": "Alice", "active": True, "age": 25}, {"name": "Bob", "active": False, "age": 30}, {"name": "Charlie", "active": True, "age": 17}, {"name": "Diana", "active": True, "age": 22}, ] print("Active adult users:") for user in users: # Skip inactive users if not user["active"]: continue # Skip minors if user["age"] < 18: continue # Only active adults reach here print(f" {user['name']}, age {user['age']}")

Break vs Continue Comparison

# ==================== SIDE BY SIDE COMPARISON ==================== print("=== With break ===") for i in range(1, 6): if i == 3: break print(i) # Output: 1, 2 (stops at 3) print("\n=== With continue ===") for i in range(1, 6): if i == 3: continue print(i) # Output: 1, 2, 4, 5 (skips 3, continues loop) # ==================== WHEN TO USE WHICH ==================== # BREAK when: # - You found what you were searching for # - An error occurred and you can't continue # - A condition means remaining iterations are useless # - You want to exit a while True loop # CONTINUE when: # - This specific item should be skipped # - You're filtering within a loop # - You want to avoid deep nesting in the loop body # ==================== AVOID DEEP NESTING WITH CONTINUE ==================== # Without continue (deep nesting): for user in users: if user["active"]: if user["age"] >= 18: if user["name"]: # Finally, the actual logic process(user) # With continue (flat structure): for user in users: if not user["active"]: continue if user["age"] < 18: continue if not user["name"]: continue # Clean, readable - only the happy path here process(user)

üí° Loop Control Best Practices

  • Use break sparingly: Too many breaks make flow hard to follow
  • Use continue for filtering: It's cleaner than nested ifs
  • Document why: Add a comment explaining why you're breaking/continuing
  • Consider functions: return can replace break in many cases
  • Use for-else: The else clause with loops is powerful for search patterns
