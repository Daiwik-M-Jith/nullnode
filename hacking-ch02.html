1. Networking: The Nervous System

The internet is a web of pipes. If you want to break into a house, you need to know where the doors, windows, and vents are.

IP Addressing & NAT

Every device needs an address. But there are two types:

  • Public IP: Your house's address on the internet (e.g., 8.8.8.8). Unique worldwide.
  • Private IP: Your laptop's address inside your house (e.g., 192.168.1.5). Not unique. Millions of people use this same IP at home.
  • NAT (Network Address Translation): The Router's job. It translates your private requests into public ones. To the outside world, all devices in your house look like ONE IP.
  • Localhost: 127.0.0.1. This means "Me". "There's no place like 127.0.0.1".

Subnets & CIDR (/24)

Networks are split into smaller chunks called Subnets. You'll see this notation everywhere: 192.168.1.0/24.

  • /24: Means the first 3 parts (192.168.1) are fixed. The last part changes.
  • Range: 192.168.1.1 to 192.168.1.254. (254 usable hosts).
  • Why hackers care: When you scan a network, you scan the subnet. "I'll scan 10.0.0.0/8" means scanning millions of IPs (loud). "I'll scan 192.168.1.0/24" means scanning 254 IPs (quiet).

The Top 10 Ports

Memorize these. You will see them every day.

21: FTP (File Transfer)
22: SSH (Secure Shell)
23: Telnet (Insecure SSH)
25: SMTP (Email Sending)
53: DNS (Domain Names)
80: HTTP (Web)
110: POP3 (Email Receiving)
139/445: SMB (Windows Shares)
443: HTTPS (Secure Web)
3306: MySQL (Database)

2. The Web: The Face of the Internet

Anatomy of a URL

https://www.example.com:8080/path/to/file?search=hacking#section2
  • Protocol: https:// (How we talk)
  • Domain: www.example.com (Who we talk to)
  • Port: :8080 (Which door to knock on - usually hidden)
  • Path: /path/to/file (Where the file is)
  • Query: ?search=hacking (Parameters sent to the server - Attack Vector!)
  • Fragment: #section2 (Scroll position - stays in browser)

Cookies & Sessions

HTTP is Stateless. The server forgets you immediately after sending a page.
To remember you (keep you logged in), we use Cookies.

  • Cookie: A small text file the server gives your browser. "Here, hold this ID card."
  • Session ID: The random number on that ID card.
  • Session Hijacking: If a hacker steals your Session ID (Cookie), they become you. They don't need your password.

3. Linux & OS: The Soul

You cannot hack if you cannot use Linux. It is the OS of the internet.

The File System Hierarchy

Windows has C:\. Linux has / (Root).

  • /bin: Binaries (Programs like ls, cat, ping).
  • /etc: Configuration files (Where the secrets live). /etc/passwd, /etc/shadow.
  • /home: User folders (like C:\Users).
  • /var: Variable data (Logs, Websites). /var/log, /var/www/html.
  • /tmp: Temporary files (Often used by hackers to store scripts because anyone can write here).

The Power of the Shell

Linux tools are like Lego bricks. You connect them with Pipes (|).

cat access.log | grep "password" | sort | uniq

1. Read the log.
2. Filter for lines containing "password".
3. Sort them alphabetically.
4. Remove duplicates.
Redirection (>): echo "hacked" > index.html (Overwrites the file).

4. Crypto & Encoding: The Code

CRITICAL: Encoding is NOT Encryption.

Encoding (No Key)

Changing data format so computers can read it. Anyone can decode it.
Base64: Ends with =. Used for email attachments and basic auth. SGVsbG8=
URL Encoding: %20 for space. %3C for <.
Hex: 48 65 6c 6c 6f.

Hashing (One-Way)

A fingerprint. Used for integrity and passwords.
MD5: Old, broken. Fast.
SHA-256: Standard. Secure.
Bcrypt: Slow. Designed for passwords.

Encryption (Requires Key)

Scrambling data for confidentiality.
AES: Symmetric (Same key). Used for WiFi (WPA2) and File Encryption.
RSA: Asymmetric (Public/Private). Used for SSL/TLS and SSH.

5. Security Theory: The Mindset

The CIA Triad

The three goals of security.
Confidentiality: Only authorized people can see it. (Encryption).
Integrity: The data hasn't been changed. (Hashing).
Availability: The system is up and running. (DDoS protection).

AuthN vs AuthZ

Authentication (AuthN): "Who are you?" (Password, Fingerprint).
Authorization (AuthZ): "What are you allowed to do?" (Admin vs User).
Broken Access Control (hacking AuthZ) is the #1 vulnerability on the web.
