From Power Button to Pwn

Your Cybersecurity Journey Roadmap

01

The Fundamentals

Think of a computer like a small town: files are houses, processes are people doing jobs, and permissions are the locks on doors. If you can't picture what's behind the door, you can't secure it.

  • Operating Systems: Learn how the 'town manager' (the OS) schedules work, manages memory, and keeps order.
  • Networking Basics: Packets are letters — understand addresses, routing and how mail moves between places.
  • Programming: Code is a recipe — learn to read and write it so you can automate and inspect behavior.
  • Command Line: The terminal is your control panel — practice simple commands so they become second nature.
  • Hardware Basics: Know where data lives and how the CPU, RAM and disk interact — this explains many bugs and vulnerabilities.

Cartoon: imagine opening the wrong mailbox and finding a password — that’s security gone wrong. Learn how mail routing and locks work first.

02

Linux Mastery

Linux runs many servers and tools — mastering it is like learning the language of those systems. This chapter focuses on the parts you'll use every day.

  • Files & Permissions: Learn what the permission bits mean and why chmod 777 is almost never a solution.
  • Package management: Installing software is how you get tools — understand repos and updates.
  • Process & services: Know how to find, inspect and control what's running (ps, top, systemctl).
  • Scripting: Small shell scripts automate boring tasks — they’ll save your time during investigations.
  • Lab-focused setup: Build a VM and practice safely — don't touch production systems.

Cartoon: think of Linux like a friendly command center — the console gives you superpowers, but one wrong switch can break the power grid.

03

Networking Deep Dive

Networks are how systems talk — learning to read those conversations is key. We'll demystify packets, ports and protocols with playful metaphors.

  • Packets = letters: IPs are addresses, ports are mailbox slots. Know how to read headers and payloads.
  • Transport protocols: TCP is a phone call (guaranteed), UDP is a shout (fast but unreliable).
  • Tools: Wireshark lets you watch traffic, nmap maps the network, netcat connects to ports.
  • Architectures: Firewalls, VLANs and VPNs shape what traffic can move where — learn why policies exist.

Cartoon: picture packets as homing pigeons — read their address tag and decide whether to let them in.

Explore networking through easy examples →
04

Programming & Scripting

Code turns ideas into actions — learn to script repetitive tasks, parse output, and prototype tools. We'll keep examples short and practical.

  • Python first: Fast to learn and powerful for automation, parsing logs, and network scripting.
  • Shell/Bash: Glue code for Linux — chaining commands saves time in investigations.
  • PowerShell: The equivalent power on Windows, useful for enterprise work.
  • Low-level languages: C and Assembly help explain memory bugs and exploit primitives — only after basics are solid.

Cartoon: think of scripts as tiny robots following a recipe — teach them simple steps before building a machine.

Practice simple scripts and automation →
05

Web Application Security

Web apps are services receiving requests — if you don't validate inputs, anyone can send unexpected data. We'll use simple stories and requests to make attacks easy to understand.

  • OWASP Top 10: These are the common failure modes (input validation, auth flaws, injection).
  • SQLi: When a form directly talks to the database — we show how input becomes queries.
  • XSS: User content shown in pages without sanitization — scripts can be injected.
  • Authentication & sessions: Understand tokens, cookies, and how they can be stolen or forged.
  • Tools: Burp Suite & ZAP are your translators — they let you look under the hood of requests.

Cartoon: imagine a form where users write messages — if you let everyone write HTML, a troll could put a hidden trap in your page.

06

Exploitation & Post-Exploitation

Exploitation is like finding a weak door and learning how to open it. Post-exploitation teaches how attackers move around and defend themselves once inside — we always practice in safe labs.

  • Recon & proof-of-concept: Learn how to find a vulnerability and confirm it without causing damage.
  • Memory bugs: Buffer overflows and use-after-free are low-level problems that let an attacker change program behavior.
  • Privilege escalation: From limited user to admin — learn the common paths and mitigations.
  • Lateral movement & persistence: Explore how attackers move and stay and what defenders watch for.

Cartoon: exploit = a crowbar to open a stuck door (but only in a target you own).

Begin safe exploitation basics →
07

Reverse Engineering

Reverse engineering tears apart a program to understand what it does without source code. We teach approachable steps — from reading strings to using debuggers.

  • Strings & clues: Sometimes a binary gives away useful clues in plain text.
  • Disassemblers & debuggers: Learn to step through code and inspect data at runtime.
  • Formats & context: Understanding PE/ELF headers helps you know how the program is structured.
  • Malware analysis: Learn safe sandbox techniques for inspecting suspicious files.

Cartoon: like reading a secret map — with patience you uncover how the treasure (or bug) works.

08

Active Directory & Enterprise Security

Enterprise identity systems like Active Directory are the backbone of big networks. Here you'll learn identity, trust, and how attackers abuse misconfigurations.

  • AD fundamentals: Users, groups, policies and how they control access across many computers.
  • Kerberos basics: Learn authentication flows and how tickets can be misused.
  • Mapping with tools: BloodHound shows attack paths conceptually — learn to read the map, not to exploit real networks.
  • Defensive thinking: Learn how to harden AD rather than only the offensive techniques.

Cartoon: AD is the office building with keycards, badges, and a front desk — know who can go where and why.

Explore enterprise security with safe simulations →
09

Cloud Security

Cloud platforms change who controls infrastructure. Learn IAM, storage, and how small mistakes (public buckets, weak permissions) cause big leaks.

  • IAM & roles: Who can do what? Least privilege is key. Learn attack surfaces created by over-permissive roles.
  • Storage and exposure: Misconfigured S3 or storage can leak data to the internet — learn to identify it.
  • Containers & serverless: New runtimes mean new risks — understand runtime isolation and common misconfigurations.
  • Cloud-native tooling: Learn to use provider tooling and common scanners for safety checks.

Cartoon: the cloud is like a public storage unit — anyone can peek in if you forget to lock it up.

Learn cloud security basics with clear examples →
10

Certifications & Practice

Certifications are structured milestones. Complement exams with daily practice in CTFs and lab platforms — that combination builds skill fast.

  • Pathways: Start with practical beginner certs, then progress to hands-on exams like the OSCP that test applied skills.
  • Practice: Use TryHackMe and HackTheBox to get real exercise on the concepts you learn.
  • CTFs & bug bounties: Apply micro-skills in challenges and develop a methodology for triaging problems.
  • Portfolio: Keep notes, solve boxes, and present writeups — this is what employers look for.

Cartoon: practice is like learning to ride a bike — start on training wheels (labs), then tackle hills (OSCP).
